# Compile-Time Profiling

Measures how long the compiler takes to evaluate `constexpr eigvals()` across different matrix sizes and categories.

## Running

From the repository root:

```sh
make profile
```

This regenerates the `compile_time/` source files, then compiles each one and records the result. The default compiler is `g++` with a 300-second timeout per file.

To use a different compiler or timeout, invoke the script directly:

```sh
./profiling/run_profiling.sh [compiler] [timeout_sec]
```

```sh
./profiling/run_profiling.sh           # g++, 300s timeout (default)
./profiling/run_profiling.sh clang++   # Clang
./profiling/run_profiling.sh g++ 600   # GCC, 10-minute timeout
```

Results are written to `profiling/results/compile_times_<compiler>_<version>.csv`.

**Requirements:** `g++` or `clang++`, `cmake`, `/usr/bin/time`, `timeout`.

## Analyzing Results

`analyze_results.py` prints a summary table and generates a plot. It runs automatically at the end of `run_profiling.sh`, but can also be run manually:

```sh
uv run profiling/analyze_results.py profiling/results/compile_times_<compiler>_<version>.csv
```

**Requirements:** Python 3.14 (see `.python-version`), managed by `uv`. `matplotlib` is optional; without it only the summary table is printed.

## Source Files

The `.cpp` files in `compile_time/` are generated by `make generate-profiling-cases` (via `octave/generate_profiling_cases.m`) and are not checked into the repository. Each file defines a single `constexpr` matrix and calls `eigvals()`, forcing full compile-time evaluation. Filenames follow the pattern `profile_<category>_<size>_<sample>.cpp`.
